Kubernetes 集群详情页 UI 方向

你选了先做 UI 重构，所以这里不引入新接口，只重排现有信息：概览、命名空间、节点、Pods、工作负载、服务、Ingress、日志入口等。

A. 概览优先推荐

入口先看健康和资源总览

顶部状态横幅 + 资源计数卡片 + 下方 Tab 表格。最接近现在结构，但视觉层级会明显提升。

适合：你希望进入详情后先知道这个集群是否正常、资源规模多大。
B. 资源浏览器清晰

左侧资源导航，右侧专注列表

把横向很多 Tab 改成左侧资源菜单，右侧是当前资源表格、筛选和状态。

适合：资源类型越来越多时，详情页更像一个小型 K8s 控制台。
C. 看板分区紧凑

概览页就是一张集群看板

把常用资源摘要、连接信息、最近状态都放到一个分区式页面，Tab 只作为补充。

适合：想少点跳转、多看摘要；但后续如果不加 Events/YAML，信息密度可能不够。
